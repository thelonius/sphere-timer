# Обновления для Backend - 19.03.2026

После последних изменений на фронтенде (реализация запланированной остановки задач и мультизадачности), необходимо внести соответствующие изменения в структуру API и базы данных.

## 1. Изменения в модели Task (Задачи)

### Новые поля:
- **`scheduled_stop_at`** (number | null): Timestamp (в миллисекундах), когда задача должна автоматически остановиться. 
  - *Логика:* Пользователь может задать время, через которое таймер выключится.
- **`is_pulsing`** (boolean): Флаг активной индикации (пульсации). 
  - *Логика:* После того как время запланированной остановки истекло, задача переходит в состояние "пульсации", пока пользователь её не "сбросит" (кликом). Это должно сохраняться при перезагрузке страницы.

### Изменение логики активации:
- **Мультизадачность:** Ранее предполагалось, что при запуске новой задачи старая должна останавливаться (`isActive = true` только у одной задачи). 
  - *Требование:* Теперь бэкенд должен разрешать **несколько одновременно активных задач** для одного пользователя. Автоматическая остановка других задач при вызове `/tasks/:id/start` должна быть отключена.

## 2. Изменения в API

### POST `/auth/refresh`
- **Описание:** Обновление JWT токена.
- **Запрос:** `{ "refreshToken": "string" }`
- **Ответ:** `{ "success": true, "data": { "token": "string" } }`
- *Примечание:* Метод уже используется в `services/api.js`, но отсутствует в старой спецификации.

### PUT `/tasks/:id`
- Должен принимать и сохранять новые поля: `scheduled_stop_at` и `is_pulsing`.

### История (History)
- В массиве `history` теперь желательно хранить поле **`timestamp`** (время завершения сессии), а не только агрегированное время за дату. Это позволит строить более точные графики в будущем.
- Текущая структура элемента истории во фронтенде:
  ```json
  {
    "date": "YYYY-MM-DD",
    "time": "number (длительность сессии в мс)",
    "timestamp": "number (время окончания сессии)"
  }
  ```

## 3. Профиль пользователя (User Profile)

### Новые поля в объекте User:
- **`language`** (string: 'ru' | 'en'): Предпочитаемый язык интерфейса.
- **`theme`** (string: 'light' | 'dark'): Предпочитаемая тема оформления.

### PUT `/user/profile`
- Должен поддерживать обновление `language` и `theme`.

## 5. Рекомендации по реализации на сервере
- Если `scheduled_stop_at` установлено, бэкенд может автоматически переводить `isActive` в `false` и обновлять `totalTime` при наступлении этого времени, чтобы данные были синхронизированы, если пользователь зашел с другого устройства.
