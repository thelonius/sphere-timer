# SphereTimer - Приложение для отслеживания времени задач

## Описание

SphereTimer - это минималистичное футуристическое веб-приложение на React для отслеживания времени, затраченного на различные задачи. Приложение помогает управлять временем и анализировать продуктивность с помощью визуальных индикаторов в виде цветных сфер.

## Возможности

🔐 **Регистрация и авторизация**:
- Регистрация новых пользователей с email и паролем
- Вход в систему
- Валидация форм
- Готовность к интеграции с бэкендом

⏱️ **Управление задачами**:
- Создание новых задач с названием и цветом
- Запуск и остановка таймера для каждой задачи
- Накопление времени при повторных запусках
- Редактирование названия и цвета задач
- Удаление задач

📊 **Календарь и статистика**:
- Просмотр времени, затраченного на задачу по дням
- Мини-календарь с визуализацией активности
- Общая статистика по задачам

🎯 **Индикатор активности**:
- Цветной кружок на экране показывает активную задачу
- Анимация пульсации для активного таймера
- Отдельное окно-индикатор для отображения поверх других окон

## Технологии

Все используемые библиотеки **бесплатные и open-source**:

- **React 19.2** - библиотека для построения UI
- **Vite 5.4** - сборщик и dev-сервер
- **CSS3** - стилизация (без дополнительных библиотек)
- **LocalStorage** - временное хранение данных (будет заменено на API)

## Установка и запуск

1. Установите зависимости (из корневой директории проекта):
```bash
npm install
```

2. Скопируйте файл с переменными окружения:
```bash
cp task-timer-app/.env.example task-timer-app/.env
```

3. Запустите приложение в режиме разработки:
```bash
npm run dev
```

4. Откройте браузер по адресу: [http://localhost:5173](http://localhost:5173)

> **Примечание:** Все команды можно запускать как из корневой директории, так и из папки `task-timer-app`

## Интеграция с бэкендом

Приложение готово к подключению бэкенда. Все API методы находятся в `src/services/api.js`.

### Настройка API

В файле `.env` укажите URL вашего бэкенда:
```env
VITE_API_URL=http://localhost:3000/api
```

### Необходимые API endpoints

**Аутентификация:**
- `POST /auth/register` - регистрация (username, email, password)
- `POST /auth/login` - вход (email, password)
- `POST /auth/logout` - выход
- `GET /auth/verify` - проверка токена

**Задачи:**
- `GET /tasks` - получить все задачи пользователя
- `POST /tasks` - создать задачу
- `PUT /tasks/:id` - обновить задачу
- `DELETE /tasks/:id` - удалить задачу
- `POST /tasks/:id/start` - начать таймер
- `POST /tasks/:id/stop` - остановить таймер
- `GET /tasks/stats` - получить статистику

**Пользователь:**
- `GET /user/profile` - получить профиль
- `PUT /user/profile` - обновить профиль
- `PUT /user/password` - изменить пароль

### Формат ответа API

Все endpoints должны возвращать JSON:
```json
{
  "success": true,
  "data": { ... },
  "message": "Success message"
}
```

При ошибках:
```json
{
  "success": false,
  "error": "Error message"
}
```

Токены передаются в заголовке:
```
Authorization: Bearer <token>
```
